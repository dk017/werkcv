# WerkCV CV Keyword Cluster Strategy

Date: 2026-03-20

This document translates the latest Dutch CV keyword set into a practical shipping plan for WerkCV.

## Goal

Win more high-intent Dutch CV traffic without creating thin duplicate pages.

Principles:

- One strong canonical page per search intent cluster.
- Route traffic to the right money page:
  - `editor` for builder intent
  - `templates` for template/style intent
  - `prijzen` for payment/trust intent
- Reuse the current 13 templates first.
- Add templates only when there is a clear conversion or coverage gap.

Examples:

- `/cv-maken-template`
- `/cv-opmaken`
- `/modern-cv-voorbeeld`
- `/mooie-cv-maken`

- Primary CTA: `Bekijk templates`

- Show 3 mapped templates above the fold

- `/cv-voorbeelden/...`
- `/stage-cv-maken`
- `/cv-maken-student`

- Primary CTA: `Gebruik dit voorbeeld in de editor`
- Secondary CTA: `Bekijk bijpassende templates`

### Comparison pages

Examples:

- `/canva-cv-alternatief`
- `/cvster-alternatief`

Use:

- Primary CTA: `Maak gratis je CV`
- Secondary CTA: `Vergelijk templates`

## Template Strategy

Current templates are enough for the next wave:

- business / default: `professional`
- ATS / formal: `ats`, `formal`
- modern: `modern`, `dynamic`
- creative / beautiful: `elegant`, `remarkable`
- student / easy / fast: `simple`, `jobboss`

Do not add templates yet unless conversion data shows a real gap.

Only consider adding later:

1. a clearer student/starter template
2. a stronger design-forward creative template
3. an extra corporate ultra-formal template

## Internal Linking Rules

Each new page should link to:

- `/editor`
- `/templates`
- `/prijzen`
- 2 to 4 related cluster pages
- at least 1 relevant example page
- at least 1 relevant template page

Examples:

- `/cv-maken-op-mobiel` should link to:
  - `/editor`
  - `/gratis-cv-maken`
  - `/cv-maken-pdf`
  - `/templates`

- `/cv-maken-student` should link to:
  - `/stage-cv-maken`
  - `/eerste-cv-maken`
  - `/cv-voorbeelden/studenten-en-starters/...`
  - `/editor`

## Content Pattern for New Pages

Each new page should include:

1. keyword-matched hero
2. short explanation of the intent
3. mapped templates or examples
4. strong CTA block
5. trust / pricing reassurance
6. FAQ
7. internal links

Avoid thin doorway pages.

## Success Metrics

Track by page cluster:

- signup rate
- click-through to `/editor`
- click-through to `/templates`
- click-through to `/prijzen`
- first paid order

If a page gets traffic but weak editor CTR, fix CTA and template mapping before creating more pages.
